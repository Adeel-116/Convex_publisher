1. Java Fundamentals & OOP Concepts

Variables & Data Types

  • Variables: Named memory locations that store data values
  • Variable Naming Rules: A-Z, a-z, 0-9 (not at start), underscore (_), $ symbol
  • Primitive Data Types:
    • byte: 8-bit integer (-128 to 127)
    • short: 16-bit integer (-32,768 to 32,767)
    • char: 16-bit Unicode character (0 to 65,535)
    • int: 32-bit integer (-2^31 to 2^31-1)
    • long: 64-bit integer (-2^63 to 2^63-1)
    • float: 32-bit floating point
    • double: 64-bit floating point
  • Type Casting:
    • Implicit (automatic): Converting smaller type to larger type
    • Explicit (manual): Converting larger type to smaller type using (type)

Operators

  1. Arithmetic: +, -, *, /, % (mathematical operations)
  2. Assignment: =, +=, -=, *=, /=, %= (assign values to variables)
  3. Comparison: >, <, <=, >=, ==, != (compare values)
  4. Logical: &&, ||, ! (combine boolean expressions)
  5. Binary: &, |, ^ (bitwise operations)
  6. Unary: ++, -- (increment/decrement)
  7. Ternary: ? : (shorthand for if-else)

OOP Concepts

  • Classes and Objects:
    • Class: Blueprint defining properties and behaviors
    • Object: Instance of a class
  • Method Overloading: Multiple methods with same name but different parameters in same class
  • Inheritance: Mechanism where a child class inherits properties/methods from parent class
    • Extends keyword used for inheritance
    • Promotes code reuse and establishes is-a relationship
  • Encapsulation: Data hiding through access modifiers
    • Private: Accessible only within the class
    • Protected: Accessible within package and subclasses
    • Public: Accessible from anywhere
  • Polymorphism:
    • Method Overriding: Child class providing specific implementation of parent's method
    • Interface Implementation: Class implementing interface methods

2. Android Architecture & Project Structure

Android Architecture (4-Layer)

  • Linux Kernel: Core services (hardware drivers, memory management, process management, security)
  • Libraries & Runtime:
    • Native C/C++ libraries for core functionality
    • Android Runtime (ART) executes Dalvik bytecode
  • Application Framework: Java API framework with manager classes
    • Activity Manager, Window Manager, Content Providers, Resource Manager, etc.
  • Applications: User-installed and system apps (top layer)

Project Structure

  • manifests/: Contains AndroidManifest.xml
    • Declares app components, permissions, features, and requirements
  • java/: Contains Java/Kotlin source code
    • Application logic, activities, fragments, services
  • res/: Contains resources (separate from code)
    • layout/: XML layout files (UI definitions)
    • values/: strings.xml, colors.xml, styles.xml (text, colors, themes)
    • drawable/: Images, shapes, and XML drawables
    • mipmap/: App icons in different densities
    • raw/: Raw files like audio, video
    • anim/: Animation definitions

3. UI Components & Layouts

Layouts (ViewGroups)

  1. LinearLayout: Arranges elements sequentially
    • orientation: vertical/horizontal determines arrangement direction
    • weight: distributes space proportionally among children
    <LinearLayout
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:orientation="vertical">
    <!-- Child views arranged vertically -->
</LinearLayout>
  2. RelativeLayout: Positions elements relative to parent or siblings
    • Elements positioned using rules (below, toRightOf, alignParentBottom, etc.)
    • More complex than LinearLayout but flexible for complex UIs
    <RelativeLayout
    android:layout_width="match_parent"
    android:layout_height="match_parent">
    <TextView
        android:id="@+id/text1"
        android:layout_below="@id/text2" />
</RelativeLayout>
  3. ConstraintLayout: Modern layout with constraint-based positioning
    • Flat view hierarchy (better performance)
    • Responsive to different screen sizes
    • Constraint handles define relationships between views
    <androidx.constraintlayout.widget.ConstraintLayout
    android:layout_width="match_parent"
    android:layout_height="match_parent">
    <!-- Views positioned with constraints -->
</androidx.constraintlayout.widget.ConstraintLayout>

View Components (UI Elements)

  1. TextView: Display text
    • Non-editable text display
    • Supports formatting, styling, and text appearance customization
    <TextView
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Hello World"
    android:textSize="20sp"
    android:textColor="#000000" />
  2. EditText: Input field
    • Subclass of TextView that allows editing
    • Different input types (text, number, password, email, etc.)
    <EditText
    android:id="@+id/editText"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:hint="Enter text here"
    android:inputType="text" />
  3. Button: Clickable button
    • Triggers actions when clicked
    • Can be customized with text, background, style
    <Button
    android:id="@+id/button"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Click Me" />

Common Attributes (Properties)

  • layout_width/height: Size dimensions
    • match_parent: Fill parent container
    • wrap_content: Size to content
    • dp values: Fixed dimensions
  • id: Unique identifier (@+id/name) for referencing in Java/XML
  • padding: Space inside the view (internal)
  • margin: Space outside the view (external)
  • visibility: visible, invisible (takes space), gone (no space)
  • textSize: Text size in sp units (scale-independent pixels)
  • textColor: Text color in hex (#RRGGBB)
  • background: Background color/drawable

4. View Binding & Event Handling

View Binding (Connecting UI to Code)

Traditional findViewById method

// In onCreate() method
Button submitButton = findViewById(R.id.submit_button);
EditText nameField = findViewById(R.id.name_field);

Event Handling

  • Responding to user interactions through listeners
  • Common events: click, long-click, touch, focus change

Click Listeners (Most Common)

submitButton.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View v) {
        String name = nameField.getText().toString();
        // Process the input
    }
});

Toast Messages (Simple Feedback)

  • Brief, non-interactive popup messages
  • Duration: LENGTH_SHORT (2 seconds) or LENGTH_LONG (3.5 seconds)
Toast.makeText(MainActivity.this, "Your message here", Toast.LENGTH_SHORT).show();

5. String Resources & Hardcode vs Softcode

Hardcoded Values (Not Recommended)

  • Values directly embedded in code or layout
  • Problems: difficult to update, doesn't support localization
<TextView
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Hello World" />

String Resources (Softcoded - Recommended)

  • Values defined in resource files
  • Benefits: centralized management, localization support, easier updates
  • strings.xml: Contains text values
  • dimens.xml: Dimension values
  • colors.xml: Color values

In strings.xml:

<resources>
    <string name="greeting">Hello World</string>
</resources>

In layout:

<TextView
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="@string/greeting" />

In Java:

String greeting = getString(R.string.greeting);

6. Intent & Navigation

Intent Class

  • Messaging object that communicates between components
  • Core functionality:
    • Starting activities
    • Starting services
    • Delivering broadcasts

Types of Intents

  1. Explicit Intent: Targets specific component by name
    • Used for communication within same app
    • Component is specified directly
  2. Implicit Intent: Declares general action
    • System finds compatible component
    • May show chooser if multiple options available

Starting a New Activity

// Explicit Intent
Intent intent = new Intent(MainActivity.this, SecondActivity.class);
startActivity(intent);

Passing Data Between Activities

  • Using extras (key-value pairs) for data transfer
// Sending data
Intent intent = new Intent(MainActivity.this, SecondActivity.class);
intent.putExtra("key_name", "value");
startActivity(intent);

// Receiving data in SecondActivity
String value = getIntent().getStringExtra("key_name");

Intent Filters

  • Declared in manifest to specify which implicit intents a component can handle
  • Defines actions, categories, and data types

7. Splash Screen

Purpose

  • Introductory screen displayed when app launches
  • Provides visual feedback during initial loading
  • Brand reinforcement and user experience enhancement

Creating a Splash Screen

  1. Create splash_activity.xml layout (visual design)
  2. Create SplashActivity.java (logic for timing and transition)
  3. Use Handler for delayed navigation
  4. Configure in AndroidManifest.xml as launcher activity
// SplashActivity.java
@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_splash);
    
    new Handler().postDelayed(new Runnable() {
        @Override
        public void run() {
            Intent intent = new Intent(SplashActivity.this, MainActivity.class);
            startActivity(intent);
            finish(); // Close SplashActivity to prevent going back
        }
    }, 3000); // 3 seconds delay
}

AndroidManifest.xml Configuration

  • Sets SplashActivity as entry point with LAUNCHER category
<activity
    android:name=".SplashActivity"
    android:exported="true">
    <intent-filter>
        <action android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />
    </intent-filter>
</activity>
